# PR #53 changelog fragment

## Added

- Network-scoped Xaman payload creation for XRPL Testnet and Mainnet.
- Canonical Mainnet XRP and official RLUSD transaction construction.
- Durable wallet-handoff network and Asset identity fields.
- Mainnet and cross-network rejection tests.

## Changed

- Xaman requests now force the Payment Intent network explicitly.
- The Provider reparses Payment Intents and revalidates exact Asset registry identity immediately before payload creation.
- The legacy Testnet payload helper now rejects Mainnet slots explicitly.

## Safety

- Mainnet payload creation requires explicit Mainnet gate access and a dedicated Mainnet Source Tag.
- Testnet and Mainnet Payment Intents cannot cross Provider boundaries.
- Unknown Assets, arbitrary IOUs, and modified XRP or RLUSD descriptors fail closed.
- Provider lifecycle status remains wallet-handoff metadata and is not treated as validated payment proof.
- The overall Mainnet Gate remains blocked pending independent verifier and operational checks.
