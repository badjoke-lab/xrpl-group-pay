# PR #59 changelog fragment

## Added

- A protected Mainnet D1 provisioning policy with immutable production and preview database names, allowed location hints, exact confirmation phrases, required tables, and required verified-payment columns.
- A fail-closed provisioner that can inspect or create the isolated production/preview D1 pair, apply every repository migration to both databases, and validate the resulting schema.
- A manually dispatched GitHub Actions workflow with default-branch enforcement, serialized execution, read-only repository permissions, and a protected provisioning environment.
- A public-safe provisioning report containing database IDs, migration counts, schema results, and a candidate D1 evidence patch.
- Unit coverage for confirmation enforcement, protected-name collisions, distinct IDs, safe Wrangler defaults, migration discovery, schema requirements, and evidence generation.

## Safety boundaries

- The workflow does not commit generated IDs, update acceptance records, deploy the Worker, or enable Mainnet runtime.
- Production and preview use separate D1 databases.
- Existing protected-name databases with unexpected tables are rejected.
- API credentials remain GitHub secrets and are not included in the report artifact.

## Release state

- This pull request adds the provisioning mechanism but does not execute production provisioning.
- The committed D1 identifiers remain placeholders.
- `production-d1-provisioning` remains pending and Mainnet release remains blocked.
