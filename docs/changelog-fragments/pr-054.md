# PR #54 changelog fragment

## Added

- A network-scoped XRPL transaction reader for Testnet and Mainnet.
- Explicit Mainnet Gate approval for Mainnet transaction reads.
- Mainnet XRP verification proofs and durable Mainnet XRP receipts.
- Mainnet XRP verification tests and cross-network rejection tests.

## Changed

- XRP proofs and receipts now preserve their XRPL network.
- XRP idempotency keys use `<network>:<transactionId>`.
- Payment verification selects the XRPL reader from the frozen PaymentSlot network and rejects deployment-network crossover.
- Pending transaction messages identify the selected XRPL network.

## Safety

- Mainnet XRP is accepted only after validated-ledger `tesSUCCESS` and exact payer, destination, amount, delivered amount, Source Tag, Destination Tag, and InvoiceID checks.
- Partial Payments, cross-currency fields, issued Amounts, malformed responses, and unavailable nodes fail closed.
- Mainnet RLUSD verification remains explicitly blocked until PR #55.
- The overall Mainnet Gate remains blocked.
