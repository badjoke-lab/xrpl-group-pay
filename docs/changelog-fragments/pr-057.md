# PR #57 changelog fragment

## Added

- A machine-readable Mainnet acceptance audit record.
- A release-decision checker that cross-validates acceptance controls, blocking findings, the Mainnet Gate, and committed safe defaults.
- Mainnet-ready deployment enforcement through `--require-ready`.

- CI and normal builds validate acceptance-record consistency without treating an intentionally blocked release as a test failure.
- The `mainnet-acceptance-audit` gate check records a failed release decision rather than remaining pending.

- Production D1 provisioning, release configuration, provider attestation, assigned Source Tag, live XRP and RLUSD evidence, and an operational stop drill remain unresolved.
