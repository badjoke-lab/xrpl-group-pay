# PR #35 changelog fragment

## Added

- A generic `verified_payment_records` table for XRP and issued Assets.
- Backfill of existing XRP receipts without inventing issued-asset drops values.
- An idempotent recorder for versioned `VerifiedPayment` contracts and canonical digests.

## Compatibility

- The existing XRP receipt table and public receipt response remain unchanged.
- Existing XRP records retain their legacy proof digest and can be upgraded with a canonical verified-payment digest.
- RLUSD slot settlement remains disabled until the asset-aware settlement switch is merged.
