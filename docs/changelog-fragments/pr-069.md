## Added

- A guarded production Xaman provider attestation runner.
- A manual GitHub Actions workflow protected by the `mainnet-xaman-attestation` Environment.
- Authenticated application metadata checks for enabled state and exact callback configuration.

## Safety boundaries

- The probe uses `SignIn` with `submit: false` and cannot create a payment.
- Raw credentials, callback paths, payload IDs, QR data, deeplinks, and WebSocket URLs are excluded from the artifact.
- The workflow has read-only repository permissions and does not deploy or modify evidence automatically.
- Mainnet runtime, gate approval, Source Tag approval, release mode, and payment operations remain closed.
