# PR #40 changelog fragment

## Added

- Asset-aware participant payment details with the frozen Asset descriptor and fixed-precision settlement amount.
- Official RLUSD issuer presentation and an explicit XRP network-fee notice in payer review and final confirmation.

## Changed

- Participant and payload responses now use one canonical payment-details contract for XRP and issued Assets.
- The existing XRP `amountDrops` field remains available only as a native-XRP compatibility field; issued Assets return it as `null`.
- Existing legacy XRP detail responses are normalized at the browser boundary so current links and tests remain compatible.

## Safety

- An issued Asset cannot be presented as XRP or accepted with a native-drops compatibility value.
- Asset network, amount code, amount scale, and legacy compatibility fields are validated together before rendering.
