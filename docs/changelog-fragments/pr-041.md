# PR #41 changelog fragment

## Added

- Browser acceptance of canonical issued-asset verification responses from the shared payment verification endpoint.
- A unified verified-payment UI state for XRP and official RLUSD.
- RLUSD verified-result coverage showing the exact delivered amount and durable receipt status.

## Changed

- Legacy XRP proofs are normalized into the versioned `VerifiedPayment` contract before rendering.
- Verified results now show the human-readable delivered Asset amount instead of an XRP-drops-only message.
- Rejected and expired Wallet Handoffs use Asset-neutral copy.

## Safety

- The browser compares the verified Asset, requested amount, payer, recipient, tags, InvoiceID, and network with the frozen PaymentSlot response before showing success.
- A structurally valid response that differs from the frozen participant conditions is shown as a verification failure.
