# PR #39 changelog fragment

## Added

- Official RLUSD Testnet Wallet Handoff support in the Xaman Provider adapter.
- Issued-asset XRPL Payment payload creation with the exact currency, issuer, decimal value, tags, and InvoiceID from the frozen Payment Intent.

## Changed

- Persisted slot request creation now consumes the provider-neutral Wallet Handoff contract instead of rebuilding an XRP-only Xaman request.
- The existing XRP Testnet request and participant response shape remain unchanged.

## Boundary

- Xaman Mainnet handoffs remain disabled.
- Participant payment details and the visible payer interface remain XRP-only until the later Asset presentation switch.
