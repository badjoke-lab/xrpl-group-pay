# PR #44 changelog fragment

## Added

- A server-authoritative fixed-precision allocation engine for Equal, Percentage, Shares, and Custom Amount strategies.
- Explicit deterministic remainder assignment to the creator, first participant, selected participant, or a validated manual distribution.
- Exact-total, unique-participant, positive-obligation, percentage-total, and share-weight validation.

## Safety

- Allocation arithmetic uses canonical integer unit strings and `bigint`; JavaScript floating-point values are not used.
- Computed remainder units cannot be hidden or silently discarded.
- Final participant obligations and creator share must equal the Bill total exactly before the result can be frozen.
