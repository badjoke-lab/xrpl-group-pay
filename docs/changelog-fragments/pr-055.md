# PR #55 changelog fragment

## Added

- A canonical RLUSD verifier for XRPL Testnet and Mainnet.
- Mainnet RLUSD verification and settlement coverage.
- Arbitrary IOU and cross-network issuer rejection tests.

## Changed

- The issued-payment strategy now routes through the canonical RLUSD verifier.
- Official Mainnet RLUSD can produce a network-scoped verified payment and durable issued-Asset receipt.
- The Mainnet RLUSD verifier gate is marked passed.

## Safety

- The verifier compares the complete Payment Intent Asset descriptor with the canonical network registry entry before ledger verification.
- Requested and delivered currency, issuer, and amount must match official RLUSD exactly.
- Partial Payments, path payments, native XRP shapes, arbitrary IOUs, and Testnet/Mainnet issuer crossover fail closed.
- Mainnet remains blocked pending operational safeguards and the final acceptance audit.
