# PR #56 changelog fragment

## Added

- A fail-closed `MAINNET_OPERATIONS_MODE` runtime control.
- `enabled`, `verify-only`, and `halted` Mainnet operations modes.
- A no-store payment operations status endpoint.
- Mainnet operational resolver, route, build-environment, and status tests.

## Changed

- Mainnet handoff creation checks the operational mode before D1 or Xaman access.
- Mainnet verification checks the operational mode before D1, Xaman, XRPL node, or settlement access.
- Mainnet builds require an explicit operations mode.
- The committed Mainnet deployment defaults to `halted`.
- The operational kill-switch Mainnet Gate check is marked passed.

## Safety

- Missing Mainnet mode defaults to a full halt at runtime.
- `verify-only` blocks new handoffs while allowing existing submitted requests to settle.
- `halted` blocks both creation and verification.
- Invalid configuration reports both operations as disabled.
- The overall Mainnet Gate remains blocked pending the final acceptance audit.
