# PR #37 changelog fragment

## Added

- Atomic PaymentSlot settlement for verified XRPL issued-asset payments.
- Exact slot checks for Asset identity, generic amount units, payer, destination, tags, InvoiceID, transaction, and receipt digest.

## Changed

- `payment_slots.paid_receipt_id` now references the generic verified-payment record table so native and issued Assets share one durable settlement boundary.
- The migration preserves existing PaymentSlots, wallet handoffs, XRP receipt links, indexes, and asset-column compatibility behavior.

## Compatibility

- The current XRP verification endpoint and public receipt response remain unchanged.
- RLUSD wallet handoff and UI remain disabled until the later endpoint and provider-capability switch.
