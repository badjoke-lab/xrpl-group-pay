# PR #42 changelog fragment

## Added

- XRP or official RLUSD selection in the Testnet Bill creation flow.
- Fixed-precision Asset amount inputs for the Bill total, creator share, and every participant allocation.
- Asset-aware review, created-Bill sharing, and progress presentation.

## Changed

- Bill and PaymentSlot publication now freezes the selected Asset identity, issuer, scale, and generic units instead of assuming XRP.
- Legacy XRP creation input remains accepted and is normalized to the canonical Asset contract.
- Existing D1 `*_drops` columns temporarily mirror six-decimal generic units for storage compatibility; application reads use the versioned Asset columns.

## Safety

- Only registered XRPL Testnet XRP and official RLUSD Assets can be selected.
- A Bill uses exactly one Settlement Asset across all participant slots.
- Allocation must be exact in the selected Asset before review or publication.
- RLUSD issuer identity is displayed before freezing and participant network fees remain identified as XRP.
