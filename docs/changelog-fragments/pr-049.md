# PR #49 changelog fragment

## Added

- Immutable Bill-level allocation records for Custom Amount, Equal, Percentage, and Shares strategies.
- Per-participant records for original strategy input, calculated base amount, remainder increment, and final frozen obligation.
- Migration of existing Bills and PaymentSlots into versioned Custom Amount allocation records.

## Changed

- Bill publication now writes the Bill, PaymentSlots, allocation metadata, and participant allocation records in one atomic D1 batch.

## Safety

- Allocation provenance uses canonical integer unit strings and does not rely on SQLite fixed-width integer arithmetic.
- Explicit remainder kind and target are retained so the frozen result can be reconstructed and audited.
- Any allocation persistence failure prevents the Bill and its PaymentSlots from being published.
