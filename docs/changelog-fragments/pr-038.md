# PR #38 changelog fragment

## Added

- An Asset-aware verification endpoint outcome for canonical issued-asset payments and generic receipts.
- Endpoint orchestration that dispatches verified XRP to the legacy atomic settlement path and verified RLUSD to the generic issued-asset settlement path.
- Stored Payment Intent reconstruction from the frozen Asset identity and generic amount units.

## Compatibility

- The existing XRP verification response, browser client, and participant UI contract remain unchanged.
- The legacy Xaman payload path remains XRP-only; RLUSD wallet handoff and participant presentation are enabled in later changes.
- Pending and failed verification responses retain their existing HTTP status and public shape.
