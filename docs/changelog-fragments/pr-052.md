# PR #52 changelog fragment

## Added

- A validated-ledger XRPL account reader for `account_info` and paginated `account_lines` data.
- Fail-closed recipient readiness decisions for XRP and issued Assets.
- Exact RLUSD issuer, currency, authorization, freeze, and trust-line capacity checks.
- Network-specific Mainnet and Testnet account-read endpoint sets.

## Changed

- The recipient readiness Mainnet gate is marked complete while the overall Mainnet gate remains blocked.

## Safety

- Mainnet account reads require explicit Mainnet gate approval.
- Unvalidated, malformed, incomplete, paginated beyond the bound, or unavailable responses never produce a ready result.
- Required Destination Tags, Deposit Authorization, missing trust lines, frozen trust lines, missing issuer authorization, and insufficient limits block readiness.
- Trust-line decimal capacity is compared with arbitrary-precision integer arithmetic rather than floating point.
